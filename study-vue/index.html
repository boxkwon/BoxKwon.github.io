  1. Node.js 개발 - 1 - 설치 및 버전 관리

    • 로컬환경에 자바스크립트를 통해서 서버개발을 할수있는 런타임 환경을 제공한다.
    • npm으로 특정 모듈들을 로컬 환경에서 관리한다.

  2. Vue 기본설치

    cdn 사용하여 vue 실행 (새창)
    • vue.js 홈페이지 다운받아서 직접 연결, cdn을 통해 웹에 있는 vue.js파일을 연결하는 총 두가지 방법이 있다.
    • 간단히 연결할수있다는것이 장점이다.
    • 개발용 버전으로 개발하고 배포하기전에 배포용으로 변경하기도 한다.
    • cdn(Content Delivery Network) : 전세계에 분산되어져있는 서버에서 다운로드 받아서 쓰는 개념
    • new라는 키워드로 시작하는 함수 실행은 생성자 함수이며, instance가 생성된다.
    • 인수로 객체 데이터를 추가하고 프로퍼티로 엘리먼트를 선택한다.

  3. Vue 시작하기 - 1 - 선언적 렌더링

    • html에 특정 요소에 템플릿 문법으로 렌더링하고 관리할수있다.
    • 자바스크립트의 특정 데이터를 변경하면 화면에 다시 자동으로 반응적으로 렌더링하는 반응형이다.
    • v-bind: 자바스크립트의 vue에서 html에 바인딩 시키는 개념
    • jquery는 html을 직접적으로 제어했다. 속도가 느린 단점이 있다. ( html의 특정요소 찾기 -> 데이터 추출 -> 새로운 데이터를 html에 삽입하는 과정을 거침 )
    • vue는 데이터를 제어하면 html에 자동으로 렌덩링 하는 개념이다. 데이터를 가지고 html을 만드는데 특화 되어있다.
    • 예제 (새창)
      1. active 데이터 boolean으로 정의
      2. v-bind 디렉티브 적용

      3. v-bind:class

        -> vue에있는 데이터와 바인딩해서 class 속성을 제어한다.

      4. class="text" v-bind:class="{ 'active' : active }"

        -> vue에서 정의한 'active'라는 데이터가 true일때만 text 클래스에 active 클래스를 추가한다

  1. Vue 시작하기 - 2 - 조건문과 반복문

    • 엘리먼트 존재 여부를 토글하는것은 간단하다.
    • "v-xxx" 디렉티브 라고 한다.
    • 직접 엘리먼트를 선택해서 제어할 필요 없이 데이터만 변경하면 렌더링도 제어가 된다.
    • Vue 엘리먼트가 Vue에 삽입/갱신/제거될 때 자동으로 트랜지션효과를 적용할 수 있다.

    • 조건문 (새창)

      1. v-if="message"

        -> 만약에 message가 비어있으면 출력하지 않겠다 라는 의미로써도 사용한다.

    • 반복문 (새창)

      1. v-for="item in items"

        -> items라는 데이터를 반복시킬건데 이 안에 들어있는 각각의 객체데이터를 item이라는 이름으로 정의해서 반복한다

      2. {{ item.message }}

        -> item에 들어있는 특정 message를 출력하려면 item이라는 객체에 들어있는 message라는 프로퍼티를 찍어서 템플릿 문법으로 출력해주세요

      3. v-bind:key="item.id"

        -> item에 들어있는 id라는 고윳값을 키로 정의하겠다.

      4. v-for 디렉티브는 v-bind:key 속성을 함께 사용하는것이 v-for의 기본 문법이다.

  1. Vue 시작하기 - 3 - 사용자 입력 핸들링 (클릭 이벤트)

    • 사용자가 앱과 상호작용할 수 있게 하기위해 v-on 디렉티브를 사용하여 vue 인스턴스에서 메소드를 호출하는 이벤트 리스너를 추가한다.
    • this : 함수안에서 자신을 가리키고, vue 인스턴스 내에서는 vue 인스턴스를 가리킨다.
    • this.message : data에 있는 message를 의미한다.
    • DOM을 건드리지 않고 앱의 상태를 업데이트 한다.
    • form의 input과 앱 상태를 양방향으로 바인딩하는 v-model 디렉티브가 있다.

    • 이벤트 핸들링 (새창)

      1. toggleElement ()

        -> ES6의 문법 : 객체 데이터의 특정한 프로퍼티가 function keyword를 정의해서 함수를 담을때 단축해서 사용할 수 있다.

      2. this.toggle = !this.toggle

        -> 추가하되 this.toggle의 현재값에 반대되는 값을 추가하겠다.

      3. v-bind:class="{ active: toggle}"

        -> active라는 클래스를 추가하는데 toggle의 값에 따라 추가하겠다.

      4. v-on:click="toggleElement"

        -> 사용자가 클릭했을때 toggleElement함수를 실행하겠다.

      5. v-modle="message"

        -> message는 문자 데이터이고 message라는 데이터를 v-model(input)에 연결해서 양방향으로 사용한다.

        • display: flex : 정렬 방식
        • justify-content : 수평 정렬
        • align-items: 수직 정렬

  1. Vue 시작하기 - 4 - 컴포넌트를 사용한 작성 방법 .1

    • 예제 (새창)

      1. Vue.component('todo-item', {...})

        -> Vue.Componet 통해서 todo-item이라는 컴포넌트를 정의한다. 첫번째 인자로 컴퍼넌트의 이름을 넣는다.

      2. v-for="item in groceryList"

        -> groceryList 배열 데이터를 반복한다. 배열안의 객체를 item이라는 이름으로 사용한다.

      3. v-bind:todo="item"

        -> todo라는 속성으로 item이라는 이름의 객체를 사용한다.

      4. props: ['todo']

        -> props를 통해서 todo라는 커스텀 프로퍼티를 html 커스텀 속성을 추가하고 데이터를 받아와서 내부에서 사용한다.

      5. <todo-item></todo-item>

        -> todo-item 이라는 커스텀화된 태그에 template의 내용이 삽입된다.

      6. template: '<li>{{ todo.text }}</li>'

        -> todo에 들어있는 프로퍼티 text를 li태그 내부에 템플릿 문법을 통해서 출력한다.

  1. Vue 시작하기 - 5 - 컴포넌트를 사용한 작성 방법 .2

    • 예제 (새창)

      1. v-for="todo in todos"

        -> todos라는 데이터를 정의할건데 그안에서 todo라는 이름에 각각의 객체데이터를 todo라는 이름으로 활용해서 쓰겠다.

      2. v-bind:key="todo.id"

        -> 데이터를 처리할때는 v-bind를 사용한다.

      3. {{ todo.title }}

        -> 템플릿 문법으로 todo에 들어있는 title을 하나씩 출력한다.

      4. v-model="todo.done"

        -> input에 데이터를 연결할 때는 v-model을 사용하여 양방향 데이터 바인딩을 한다.

        -> todo에 들어있는 done의 데이터를 기준으로 체크 유무를 정의해서 화면에 렌더링 한다.

      5. v-bind:class="{ done: todo.done }"

        -> done이라는 클래스를 추가하는데 todo에 들어있는 done의 상태에 따라서 추가할지 결정한다

      6. template: `<li>...</li>`

        -> template을 여러번 반복시키기 때문에 내부에 v-for와 v-bind:key는 있을 필요가 없다.

      7. props: ['todo']

        -> 외부에서 todo라는 이름으로 받아와서 내부에 전달해 준다.

      8. v-bind:todo="todo"

        -> todo라는 속성을 부여하고 todo 객체를 전달한다.

    • 예제 (새창)

  1. Vue 가이드 - 02. Vue 라이프사이클 -1

    1. 기본적인 이벤트를 정의되면 beforeCreate라는 hook이 실행된다.

    2. 외부 모듈을 가져다 사용하거나 반응성을 부여하면 created라는 hook이 실행된다.

    3. el이라는 옵션이 있는지 체크한다.

    4. template이라는 옵션이 있는지 체크한다.

    5. redering하거나 compile한다.

    6. 연결이 되기 전에 beforeMount가 실행된다.

    7. 연결이 되면 mounted가 실행된다.

    8. 데이터가 변경되어 화면이 재 렌더링 되기 전에 beforeUpdate가 실행된다

    9. 화면이 재 렌더링 된 후에 updated가 실행된다.

    10. 인스턴스가 더이상 필요없을때 $destroy()를 실행하여 파괴하면 beforeDestroy가 실행된다.

    11. vue인스턴스의 기본적인 내용들이 파괴되면 destroyed가 실행된다.

  1. Vue 가이드 - 03. Vue 라이프사이클 -2

    • 예제 (새창)

      1. beforeCreate({console.log(this.message)})

        -> vue 인스턴스가 생성되기 전이기 때문에 이터에 있는 message가 무엇인지 모른다. undefined가 뜬다.

      2. created({console.log(this.message)})

        -> 인스턴스가 생성된 나서 실행되기 때문에 Hello Vue가 출력된다.

      3. vue 에서 제공하는 ref속성을 사용하여 "div"라는 이름의 요소를 참조하겠다.

      4. beforeMount({console.log(this.$refs.div)})

        -> ref속성으로 정의한 각각의 요소들을 가지고있는데 그 안에서 "div"라는 이름의 요소를 참조하여 보여주겠다.

        -> html요소와 vue 인스턴스가 연결되기 전이라 undefined가 뜬다.

      5. mounted({console.log(this.$refs.div)})

        -> 연결된 후에 체크하기때문에 "div"라는 이름의 요소를 출력한다.

      6. vue 에서 제공하는 ref속성을 사용하여 "message"라는 이름의 요소를 참조하겠다.

      7. beforeUpdate({console.log(this.$refs.message.innerText)})

        -> 데이터가 변경되고 나서 화면이 재 렌더링 되기 직전에 실행되기 때문에 화면에 원래 그려져있던 text가 출력된다.

      8. updated({console.log(this.$refs.message.innerText)})

        -> 데이터가 변경되고 화면이 재 렌더링 된 후에 실행되었기 때문에 변경한 text가 화면에 출력된다.

      9. beforeDestroy()

      10. destroyed()

        -> vueModel.message = "Hello World~" 를 실행해도 데이터는 들어갔지만 반응성이 사라졌다.

        -> vue 인스턴스를 사용하지않는데 메모리를 잡아먹거나 하는 상황에서 사용하기도 한다.

  1. Vue 가이드 - 04. Vue 템플릿 문법

    • 예제 (새창)

      1. <div v-once>

        -> 반응성으로 매번 렌더링 하지않고 한번만 렌더링하고 관찰하지 않겠다

      2. <div v-html>

        -> 특정 태그를 직접적으로 삽입 해야 할때, 순수 HTML 태그를 삽입하려고 할때는 v-html 디렉티브를 사용한다.

      3. 보간법(Interpolation)에는 자바스크립트의 모든 표현식을 사용할 수 있다.

      4. <div v-bind:class="..." v-on:click="..."> -> <div :class @click>

        -> 데이터가 특정한 속성에서 사용되려면 특정 디렉티브를 붙여 주어야 한다.

  1. Vue 가이드 - 05. computed

    • 예제 (새창)

      1. computed: {...}

        -> 서버에서 데이터를 가지고와서 화면을 만들어 내려고하는데 내가 필요한 속성이 없을때 사용한다.

      2. return this.todos

        -> 특정 값을 return 받아야 한다.

        -> data 옵션 내부의 특정 데이터를 연결해 준다.

      3. return this.todos.map((todo, index => {...}))

        -> 원본 데이터 todo가 갱신되면 computedTodos도 갱신되는 환경 설계

        -> 원본 데이터를 반복한 후 가공된 데이터를 반환해 주는 메소드인 map()사용
