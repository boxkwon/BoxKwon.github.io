2021.06.22

  1. Node.js 개발 - 1 - 설치 및 버전 관리

    • 프론트앤드 개발을 하기위해서는 node.js 환경이 필요하다.
    • 로컬환경에 자바스크립트를 통해서 서버개발을 할수있는 런타임 환경을 제공한다.
    • npm으로 특정 모듈들을 로컬 환경에서 관리한다.

  2. Vue 기본설치

    cdn 사용하여 vue 실행 (새창)
    • vue.js 홈페이지 다운받아서 직접 연결, cdn을 통해 웹에 있는 vue.js파일을 연결하는 총 두가지 방법이 있다.
    • 간단히 연결할수있다는것이 장점이다.
    • 개발용 버전으로 개발하고 배포하기전에 배포용으로 변경하기도 한다.
    • cdn(Content Delivery Network) : 전세계에 분산되어져있는 서버에서 다운로드 받아서 쓰는 개념
    • new라는 키워드로 시작하는 함수 실행은 생성자 함수이며, instance가 생성된다.
    • 인수로 객체 데이터를 추가하고 프로퍼티로 엘리먼트를 선택한다.

  3. Vue 시작하기 - 1 - 선언적 렌더링

    • html에 특정 요소에 템플릿 문법으로 렌더링하고 관리할수있다.
    • 자바스크립트의 특정 데이터를 변경하면 화면에 다시 자동으로 반응적으로 렌더링하는 반응형이다.
    • v-bind: 자바스크립트의 vue에서 html에 바인딩 시키는 개념
    • jquery는 html을 직접적으로 제어했다. 속도가 느린 단점이 있다. ( html의 특정요소 찾기 -> 데이터 추출 -> 새로운 데이터를 html에 삽입하는 과정을 거침 )
    • vue는 데이터를 제어하면 html에 자동으로 렌덩링 하는 개념이다. 데이터를 가지고 html을 만드는데 특화 되어있다.
    • 예제 (새창)
      1. active 데이터 boolean으로 정의
      2. v-bind 디렉티브 적용

      3. v-bind:class

        -> vue에있는 데이터와 바인딩해서 class 속성을 제어한다.

      4. class="text" v-bind:class="{ 'active' : active }"

        -> vue에서 정의한 'active'라는 데이터가 true일때만 text 클래스에 active 클래스를 추가한다

  1. Vue 시작하기 - 2 - 조건문과 반복문

    • 엘리먼트 존재 여부를 토글하는것은 간단하다.
    • "v-xxx" 디렉티브 라고 한다.
    • 직접 엘리먼트를 선택해서 제어할 필요 없이 데이터만 변경하면 렌더링도 제어가 된다.
    • Vue 엘리먼트가 Vue에 삽입/갱신/제거될 때 자동으로 트랜지션효과를 적용할 수 있다.

    • 조건문 (새창)

      1. v-if="message"

        -> 만약에 message가 비어있으면 출력하지 않겠다 라는 의미로써도 사용한다.

    • 반복문 (새창)

      1. v-for="item in items"

        -> items라는 데이터를 반복시킬건데 이 안에 들어있는 각각의 객체데이터를 item이라는 이름으로 정의해서 반복한다

      2. {{ item.message }}

        -> item에 들어있는 특정 message를 출력하려면 item이라는 객체에 들어있는 message라는 프로퍼티를 찍어서 템플릿 문법으로 출력해주세요

      3. v-for 디렉티브는 v-bind:key 속성을 반드시 추가해 주어야한다.
      4. 각각의 아이템은 고유하게 동작해야 한다.
      5. v-for의 기본 문법이다.
      6. 각각의 객체데이터에 id라는 속성을 가지고 있고 id는 기본적으로 고유하다.
